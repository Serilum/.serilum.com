Feature Requests

Last updated (~every 6h)

In order to reduce the amount of open issues on the GitHub issue tracker, new feature requests are closed as "not planned" when created.

The wording "not planned" is chosen as a way to separate the feature requests from an actual completed issue. This does not actually mean it's not planned!

To up-vote the feature you'd most like to see implemented, you can react to the issue's first post on GitHub with a positive emoji, such as: 👍 😀 🎉 💚 🚀 👀

Click any row below to open its issue page. The request with the most reactions shows at the top, so I can prioritize what users want the most. I can't implement everything, but I'll try my best!
